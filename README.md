# GrowthRoom - Employee Management System

A comprehensive Django-based employee management system designed to streamline HR operations, training, performance tracking, and attendance management.

## 🚀 Features

### Core Modules
- **Employee Management**: Complete employee profiles with personal and professional information
- **Attendance Tracking**: Daily attendance records with filtering and reporting
- **Task Management**: Project and task assignment with progress tracking
- **Performance Reviews**: Employee performance evaluation and feedback system
- **Training Sessions**: Virtual training management with employee assignments
- **Certificate Management**: Digital certificate issuance and tracking
- **Employee Portal**: Dedicated interface for employees to view their information

### Key Features
- **Role-based Access Control**: Different views for admins, staff, and regular employees
- **Training Assignment**: Assign specific training sessions to employees
- **Automatic Attendance**: Training attendance is automatically recorded when employees join sessions
- **Performance Tracking**: Link performance reviews to specific tasks
- **Modern UI**: Bootstrap 5 based responsive design
- **Admin Interface**: Comprehensive Django admin for all modules

## 🛠️ Technology Stack

- **Backend**: Django 4.2.10
- **Database**: PostgreSQL (Production) / SQLite (Development)
- **Frontend**: Bootstrap 5, HTML5, CSS3
- **Forms**: Django Crispy Forms
- **Tables**: Django Tables2
- **Filtering**: Django Filter
- **Deployment**: Render (with Blueprint)

## 📋 Prerequisites

- Python 3.12+

## 🔧 Local Development Setup

1. **Clone the repository**
   ```bash
   git clone https://github.com/gangeshwar01/growthroom.git
   cd growthroom
   ```

2. **Create and activate virtual environment**
   ```bash
   python -m venv venv
   # On Windows
   venv\Scripts\activate
   # On macOS/Linux
   source venv/bin/activate
   ```

3. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

4. **Run migrations**
   ```bash
   python manage.py migrate
   ```

5. **Create superuser**
   ```bash
   python manage.py createsuperuser
   ```

6. **Run development server**
   ```bash
   python manage.py runserver
   ```

7. **Access the application**
   - Main application: http://127.0.0.1:8000/
   - Admin interface: http://127.0.0.1:8000/admin/
   - Employee portal: http://127.0.0.1:8000/employee-portal/

## 🚀 Deployment on Render

### Automatic Deployment (Recommended)

1. **Fork/Clone this repository** to your GitHub account
2. **Sign up/Login** to [Render.com](https://render.com)
3. **Create a new Blueprint** and connect your GitHub repository
4. **Render will automatically detect** the `render.yaml` file and set up:
   - PostgreSQL database
   - Web service with Gunicorn
   - Environment variables
   - Automatic deployments

### Manual Deployment

1. **Create a new Web Service** on Render
2. **Connect your GitHub repository**
3. **Configure the service**:
   - **Build Command**: `./build.sh`
   - **Start Command**: `gunicorn growthroom.wsgi:application`
   - **Python Version**: 3.12

4. **Add Environment Variables**:
   - `SECRET_KEY`: Auto-generated by Render
   - `DATABASE_URL`: Auto-linked from PostgreSQL service
   - `DEBUG`: Set to `0` for production

## 🔧 Configuration

### Environment Variables

- `SECRET_KEY`: Django secret key
- `DEBUG`: Debug mode (0 for production, 1 for development)
- `DATABASE_URL`: Database connection string
- `ALLOWED_HOSTS`: Comma-separated list of allowed hosts

### Database Configuration

The application automatically configures the database based on the `DATABASE_URL` environment variable:
- **Development**: SQLite database
- **Production**: PostgreSQL (via Render)
